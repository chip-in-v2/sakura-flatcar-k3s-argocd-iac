# CNI

CNIにはCiliumを使用する。

## 割り当てIPによるアクセス

pod ネットワークにおいて、pod が他のpod のサービスに接続する場合は名前ではなくIPアドレスを指定してアクセスする。ただし、実際の pod ネットワークのアドレスではなく、別途割り当てた実態のない IPアドレスでアクセスする。
クライアントの pod が割り当てIPで connect する際に eBPF でインターセプトして宛先のIPとポート番号を実際の pod のものに書き換える。これにより、クライアント pod でDNSを呼び出して名前解決を使用せずにすむ。DNSへの登録の遅れや TTL による更新の検知遅れを機にする必要がなくなる。

## 仮想IPの付与

外部からのアクセスにおいて、pod が仮想IPを使用できるようにする。 pod が起動したワーカノードにおいて仮想IPへのアクセスを実pod へのアクセスに書き換える。ただし、さくらのクラウドのロードバランサはDSR方式であるため、 GARP パケットの送信やBGPによるルーティング制御は行わない。また、DaemonSet となるサービスの場合、すべてのワーカノードで仮想IP書き換え処理を行う。

butane/node.yaml.tpl の /etc/systemd/network/20-lo-vip.network の設定は削除する。

## ロードバランサのヘルスチェック対応

さくらのクラウドのロードバランサはtcp, http のヘルスチェックを実IP宛に行う。これに応答する必要があるので、 Cilium では仮想IPに加えて実IPに対する接続も traefik の pod に転送する。

Traefik サービスの `externalIPs` に各ノードの LB ルータネットワーク側 IP (eth0) を追加することで、Cilium の kube-proxy replacement が仮想IP (`lb_vip_ip`) と同様にノード実IP宛の接続をインターセプトして Traefik pod へ転送する。
